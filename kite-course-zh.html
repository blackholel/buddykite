📦

复杂的安装步骤让人放弃，Kite 一键安装

🤯

"太复杂了，算了吧..."

AI 工具门槛太高，Kite 把门槛降到地面

核心理念

Kite 的设计目标很简单：下载 → 打开 → 开始使用。像跟朋友聊天一样告诉 AI 你的想法，它会帮你实现。

Kite 能做什么？

🎨

"帮我设计一个简约风格的作品集网站" —— AI 写代码、创建文件、实时预览

Claude Code SDK 连接 Claude AI 的核心

02

认识主要角色

了解 Kite 的四大核心组件

应用的核心结构

Kite 像一场演出，有四个主要"演员"各司其职：

src/main/ 主进程 —— 应用的大脑
index.ts 程序入口，创建窗口
services/ 各种服务（配置、AI、对话等）
ipc/ 进程间通信处理器
src/renderer/ 渲染进程 —— 你看到界面
App.tsx 主应用组件
stores/ 状态管理（Zustand）
components/ UI 组件
src/preload/ 预加载脚本 —— 安全桥梁
src/shared/ 共享代码 —— 主进程和渲染进程共用

角色 1: 主进程（Main Process）

主进程是应用的"大脑"，它控制着一切底层操作。

🧠
什么是主进程？

Electron 应用运行时会启动一个主进程，它拥有完整的操作系统访问权限。窗口管理、系统菜单、文件操作等都由它处理。

主进程负责的关键任务：

1
窗口管理

创建和管理应用窗口，决定窗口的大小、位置和行为

2
系统菜单

创建应用菜单（文件、编辑、视图等）

3
AI 服务集成

与 Claude Code SDK 通信，处理 AI 对话

4
文件访问

读取和写入本地文件，这是 Web 页面做不到的

角色 2: 渲染进程（Renderer Process）

渲染进程就是你在窗口中看到的界面——所有按钮、输入框、聊天消息都在这里生成。

🎨
类比：剧院舞台

如果主进程是后台导演，渲染进程就是舞台上的演员——它直接面对观众（用户），展示一切。

渲染进程的特点：

🖥️
基于 Web 技术

使用 React、HTML、CSS 构建和渲染界面

🔒
沙箱隔离

运行在沙箱中，不能直接访问操作系统

📡
通过 IPC 与主进程通信

需要主进程帮助完成文件操作等任务

SDK 负责的核心功能：

💬

对话生成

🛠️

📡

实时推送 AI 的思考过程和工具执行

角色 4: 状态管理（Zustand）

Zustand 是一个轻量级的状态管理库，用于在组件之间共享数据。

📦
什么是状态？

状态就是应用中随时会变化的数据——比如当前聊天消息、用户设置、选中的对话等。

Kite 使用的主要状态存储：

💬

聊天消息、对话历史、当前会话

🗂️

⚙️

应用配置、主题设置、视图状态

🔧

技能管理、自定义能力

组件之间如何对话

IPC 通信与事件系统

为什么需要通信？

主进程和渲染进程运行在不同的上下文中，它们不能直接互相访问。当渲染进程需要执行文件操作或调用 AI 时，必须请求主进程帮忙。

🏦
类比：银行柜台

就像你（渲染进程）不能直接打开保险库，需要通过柜员（IPC）请求主进程帮你完成。这就是为什么需要 IPC（进程间通信）。

真实代码：消息发送

看看当用户发送消息时，实际的代码是如何工作的：

CODE 
// 渲染进程：发送消息给主进程
const response = await api.sendMessage({
  spaceId: 'workspace-1',
  conversationId: 'chat-123',
  message: '帮我写一个 hello world'
})
通俗解释

调用 API 的 sendMessage 方法

传递当前工作空间 ID，告诉 AI 在哪个项目里工作

传递当前对话 ID，标记这是哪一次对话

用户输入的具体内容——AI 需要理解的需求

IPC 处理器（主进程端）

主进程监听这些请求并处理：

CODE 
// 主进程：注册消息处理器
ipcMain.handle(
  'agent:send-message',
  async (event, request) => {
    const result = await agentController.sendMessage(
      mainWindow,
      request
    )
    return result
  }
)
通俗解释

告诉 Electron："当我收到 'agent:send-message' 请求时..."

这是一个异步处理器，等待 AI 处理完成

调用 agentController 来处理实际的 AI 逻辑

返回结果给渲染进程

消息流程服务

message-flow.service.ts 是最关键的文件之一，它管理整个对话流程：

CODE 
// 消息流程的核心循环
async processMessage(request: SendMessageRequest) {
  // 1. 验证和准备
  const session = await getOrCreateSession(...)

  // 2. 发送给 Claude
  const stream = await session.sendMessage(request.message)

  // 3. 处理响应
  for await (const event of stream) {
    await handleEvent(event, mainWindow)
  }
}
通俗解释

获取或创建一个对话会话

把用户消息发送给 Claude，得到一个"流"——数据会陆续到来

循环处理每个事件（思考、工具调用、响应等），实时更新界面

分段初始化

Kite 使用分阶段初始化，确保应用启动速度：

1

基础服务

→
2

窗口显示

→
3

⚡
为什么这样做？

用户最关心的是"能用了没"。先显示界面，再加载复杂功能，让感知速度更快。

懒加载

Kite 使用懒加载来减少初始加载时间：

CODE 
// App.tsx 中的懒加载
const SpacePage = lazy(
  () => import('./pages/SpacePage')
)

const UnifiedPage = lazy(
  () => import('./pages/UnifiedPage')
)

const SettingsPage = lazy(
  () => import('./pages/SettingsPage')
)
通俗解释

使用 React 的 lazy 函数延迟导入

这些页面组件不会在应用启动时加载

只有当用户真正访问这些页面时才会加载

大幅减少首次打开应用的时间

安全设计

Kite 采用多层安全措施保护用户：

🔒
上下文隔离

渲染进程和 Node.js 完全隔离，防止恶意网页代码

🛡️
预加载脚本桥接

通过安全的 channel 与主进程通信

📁
资源访问控制

AI 只能访问工作目录下的文件，防止越权访问

⏸️
工具执行审批

敏感操作需要用户确认才能执行
