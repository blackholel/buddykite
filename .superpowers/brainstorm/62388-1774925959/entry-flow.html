创建技能入口，主路径要定哪一条？

你现在不满意的不是弹框样式，而是用户被带离了当前工作区语境。先把默认入口定死，后面状态流才不会反复返工。

现状: 资源库页 → Create → 弹框生成 → 直接落库
技能资源库
ResourceCreateModal
问题: 脱离当前工作区和会话上下文，创建像“填表单”，不像“在工作里完成任务”。
目标: 入口触发 → 回到当前工作区 → 输入框预填 /创建技能
当前工作区会话
已有上下文、附件、历史消息仍然可见
Skill Creator
/创建技能
优势: 不丢上下文，后续追问、补充、确认、复盘都在同一条线程完成。

我建议优先选 B

A

保留资源库页 Create，但仍在本页处理

只是把弹框改成页内面板或抽屉。改动小，但没有解决“离开工作语境”这个根问题。

B

保留入口，但点击后跳回当前工作区并预填 /创建技能

这是最稳的主路径。入口还在，心智切到对话里，用户仍能从资源库页发起创建。

C

彻底移除资源库页 Create，只允许在工作区内创建

最纯，但会让资源库页少一个显式入口。对新用户不一定友好，容易让人以为这里只能管理不能创建。
