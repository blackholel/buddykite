统一工作台的信息架构

已锁定约束：只展开当前空间的最近聊天；其他空间默认折叠，只显示名称和状态。

A

推荐：单栏分组式左侧导航

一个左栏，顶部是空间切换，中部是空间列表，当前空间下面内嵌最近聊天，底部是设置。

优点

  • 最直观，学习成本最低
  • 空间和聊天关系最清楚
  • 实现成本相对可控

风险

  • 空间很多时左栏会变长
B

双层导航：顶部空间切换 + 左栏当前空间聊天

左栏不列所有空间，只显示当前空间聊天；所有空间通过顶部切换器打开抽屉或下拉查看。

优点

  • 左栏更干净
  • 主工作流更聚焦当前空间

风险

  • 空间发现性变差
  • 切换空间多一步
C

左栏空间列表 + 右侧聊天抽屉

左栏只看空间，当前空间聊天改成右侧抽屉或顶部 tab。

结构预览

空间 C

设置
主聊天工作区
当前空间聊天抽屉

优点

  • 空间管理感更强

风险

  • 布局更复杂
  • 不符合你要的“更直观”
  • 聊天作为主任务被分散

我的建议

先选 A。它最接近真实用户心智：空间是目录，对话是目录下最近打开的工作。不要过早做 B 那种隐藏式空间抽屉，也不要做 C 那种左右双轨。
