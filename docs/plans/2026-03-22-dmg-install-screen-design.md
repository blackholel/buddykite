# DMG 安装界面极简重设计

**目标**

将 macOS `dmg` 安装窗口重构为“单一主路径 + 单一说明文件兜底”的极简引导界面，让第一次安装的用户一眼知道下一步动作，避免被异常说明和不可执行的辅助脚本打断。

**问题复盘**

- 当前背景图把安装步骤、安全提示、命令排查全部堆在同一视觉层级。
- 安装说明文本文件与修复脚本的图标直接压进正文区域，造成内容遮挡和认知噪音。
- 散装 `.command` 脚本位于已下载的 DMG 中时，会先被 macOS Gatekeeper 拦截，无法承担“自动修复”的兜底职责。
- 用户真正需要完成的动作只有一个，但现有界面让人误以为要先读完一整份故障手册。

**设计方向**

- 风格：克制、清晰、轻玻璃感、接近 macOS 工具类产品的低噪声体验。
- 主路径：只强调“把 buddykite 拖到 Applications”。
- 次路径：把“首次打开遇到拦截怎么办”降级到底部说明区，只保留可阅读、可复制的手动处理说明。
- 文案原则：短句、动词优先、按动作顺序组织。

**版式方案**

1. 顶部信息区
   - 小标签：`macOS 安装`
   - 主标题：`把 buddykite 拖到 Applications`
   - 副标题：`安装只需要这一步，完成后请从 Applications 打开。`

2. 中部操作区
   - 预留左侧应用图标与右侧 `Applications` 图标位置。
   - 使用简洁箭头和细节提示强化拖拽方向，不再展示冗长说明。

3. 底部说明区
   - 三个短步骤卡片：
     - `1 拖动应用`
     - `2 从 Applications 打开`
     - `3 若被拦截，再用下方工具`
   - 单独一条轻提示：`下方文件仅在系统拦截时使用，不影响正常安装。`

4. 底栏说明区
   - 只保留安装说明文本文件入口。
   - 文本文件内提供系统设置处理路径与可复制的终端命令，不再展示不可执行的修复脚本。

**非目标**

- 不修改应用内 onboarding 或设置页。
- 不新增复杂动画或额外安装步骤。
- 不再继续分发 DMG 内的散装修复脚本。

**验收标准**

- 用户第一眼能明确识别唯一主动作。
- 背景正文不再被底部工具图标遮挡。
- 安装失败的排查路径仍然保留，但不抢占主流程注意力。
- DMG 内不再包含会被系统优先拦截的 `.command` 修复脚本。
